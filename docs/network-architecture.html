01 End-to-end code flow

One photo flows through up to five models. A car-presence gate runs first; then a make/model identifier and a damage model run in parallel; their signals feed an XGBoost cost regressor that is calibrated to a swappable parts-price catalog and converted to the chosen currency.

02 How the data is parsed

Two datasets, two parsers, one canonical pipeline. Stanford Cars trains the identifier; CarDD (COCO format) trains the damage models. Every image is decoded, cropped, augmented, tensorised and normalised to ImageNet statistics before it touches a network.

Stanford Cars → identifier

16,185 images · 196 classes ("Make Model BodyType Year")

Class names and per-image bounding boxes live in MATLAB .mat devkit files (cars_meta.mat, cars_train_annos.mat). The loader reads them with scipy.io.loadmat, crops each car to its annotation box, then applies the transform. A deterministic stratified 90/10 split makes train/val.

CarDD → damage models

~4,000 images · 6 damage types · COCO bbox + polygon masks

COCO JSON (instances_*2017.json) is parsed into canonical Records. Variant B uses normalised YOLO boxes; Variant C rasterises the polygon segmentation into per-instance masks for Mask R-CNN.

03 Network architecture

The identifier and the Variant A damage classifier share the same skeleton: an ImageNet ResNet-50 backbone (conv1 → layer1…layer4 → avgpool) with the final fully-connected layer replaced by a small custom head. The two-stage fine-tune controls which blocks receive gradients.

Custom head — identifier (196 classes)

Dropout(0.2) → Linear(2048 → 512) → ReLU → Dropout(0.3) → Linear(512 → 196) → Softmax

The 2048-d vector after avgpool is also reused as a frozen feature extractor for the XGBoost cost regressors.

Custom head — damage classifier (6 types)

Dropout(0.3) → Linear(2048 → 512) → ReLU → Dropout(0.4) → Linear(512 → 6) → Sigmoid

Multi-label: each of the 6 damage types fires independently, so the output is a sigmoid per class, not a softmax.

Variant C — Mask R-CNN (ResNet-50-FPN)

Same backbone family, wrapped in a Feature Pyramid Network and topped with a Region Proposal Network plus three heads. COCO-pretrained, then the box & mask predictors are re-initialised to 6 damage classes + background.

04 Activation functions

Four non-linearities appear across the models. Hover a plot to read its value.

ReLU hidden layers (ResNet + heads)

f(x) = max(0, x)

Sigmoid multi-label damage output

σ(x) = 1 / (1 + e−x)

Softmax make/model output (196-way)

softmax(x)i = exi / Σj exj

SiLU YOLOv8 detector (internal)

silu(x) = x · σ(x)

05 How the model is trained

The CNNs use a two-stage transfer-learning recipe; Mask R-CNN uses SGD with a step schedule and mixed precision. Below: the freeze schedule, optimisers, and the augmentation that fights overfitting on small datasets.

Optimisers

Identifier / classifier

AdamW, lr 1e-3 → 1e-4, wd 1e-4, ReduceLROnPlateau

Segmenter (Mask R-CNN)

SGD, lr 5e-3, momentum 0.9, wd 5e-4, StepLR(8, 0.1)

Regularisation

Dropout (0.2–0.4), weight decay, label smoothing 0.1, gradient clipping (‖g‖ ≤ 5.0), early stopping on val metric.

Augmentation

RandAugment(2 ops, mag 9), MixUp(α=0.2) & CutMix(α=1.0) applied per-batch with probability 0.8 — targets become soft labels.

MixUp / CutMix — what the network actually sees

Two images and their labels are blended (MixUp) or one is pasted into the other (CutMix); the loss is computed against the mixed soft target. This is why training uses a soft cross-entropy.

06 Backpropagation logic

PyTorch autograd records the forward graph; loss.backward() walks it in reverse, multiplying local gradients via the chain rule. Frozen blocks (Stage 1) get no gradient; gradients are clipped, optionally loss-scaled (AMP), and applied by the optimiser.

The chain rule, per layer

∂L/∂Wk = ∂L/∂aout · ∏ (∂ai+1/∂ai) · ∂ak/∂Wk

Stage 1 freezes layer1…layer4 (requires_grad=False) so the product stops at the head. Stage 2 unfreezes layer3+layer4 and gradients flow deeper.

07 Loss / error functions

Each task minimises a different objective. The plots show how loss grows as the prediction drifts from the truth.

Identifier — Cross-Entropy (+ label smoothing)

L = − Σc yc · log( softmax(x)c )

Damage classifier — Binary Cross-Entropy

L = − Σc [ yc log σ(xc) + (1−yc) log(1−σ(xc)) ]

Mask R-CNN — multi-task sum

L = Lrpn-cls + Lrpn-box + Lcls + Lbox + Lmask

Classification uses cross-entropy, box regression uses smooth-L1, and the mask head uses per-pixel binary cross-entropy. The trainer sums them into one scalar.

XGBoost cost — squared error

L = Σi ( ŷi − yi )²  ·  objective = reg:squarederror

The target is a synthetic, catalog-derived repair cost; the booster fits residuals stage by stage with early stopping on validation RMSE.

09 The final output & how the image is shown

The Gradio app returns an annotated image plus structured text. The original photo is redrawn with a green car-box (from the gate) and coloured damage boxes/masks; a JSON blob carries full provenance.

Annotated image

car box (gate) dent scratch glass

Returned payload

{ "car_check": { "label": "car", "score": 0.99 }, "identified": { "make": "audi", "model": "tts", "year": 2012, "confidence": 0.51 }, "damage_types": ["dent","scratch"], "parts": ["front_bumper","hood"], "cost_usd": 1284.0, "cost": 107312.0, "currency": "INR", "tier": "exact", "provenance": "xgb_b(exact); calibrated to …" }

If the identifier's confidence is below the 0.30 floor, make becomes null and the cost falls back to body-type/segment pricing instead of a wrong label.
